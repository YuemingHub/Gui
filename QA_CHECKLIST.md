# QA_CHECKLIST

## 目标

这份清单用于验证当前前端 MVP 是否满足最基本的稳定性要求：

- 能打开
- 能输入
- 能保存
- 能刷新不崩
- 导入导出可用
- 重置可用
- lint / build 通过

---

## 一、基础运行

### 1. 本地开发启动

- [ ] 运行 `npm run dev`
- [ ] 页面能正常打开
- [ ] 首屏没有白屏或明显报错
- [ ] 控制台没有 hydration error

### 2. 生产构建

- [ ] 运行 `npm run build`
- [ ] 构建成功
- [ ] 没有阻断性类型或 lint 问题

### 3. ESLint

- [ ] 运行 `npm run lint`
- [ ] lint 通过

---

## 二、基础持久化

### 4. 首次进入

- [ ] 首次进入能看到欢迎引导
- [ ] 首页能正常输入
- [ ] 任一 section 输入后页面不报错

### 5. 自动保存

- [ ] 在“回到自己”填写内容后，保存提示正常变化
- [ ] 切换 section 后内容仍保留
- [ ] 刷新页面后内容仍保留

### 6. hash section 恢复

- [ ] 切换到任一非首页 section
- [ ] 刷新后仍停留在该 section
- [ ] 没有 section 错乱或空白渲染

## 五、导出 / 导入 / 重置

### 16. 导出

- [ ] 文件名带日期
- [ ] 导出后状态提示正常

### 17. 导入有效文件

### 18. 导入无效文件

## 六、异常恢复

### 20. 损坏 localStorage

手动把存储内容改成错误结构后验证：

- [ ] 页面仍可打开
- [ ] 不会因为结构错误直接崩溃
- [ ] 能回退到默认结构或归一化结构
- [ ] 有合理提示文案

### 21. 缺字段数据

手动移除部分字段后验证：

- [ ] 页面仍可打开
- [ ] 缺失字段被自动补齐
- [ ] 各 section 不因 `undefined` 崩溃

---

## 七、界面稳定性

### 22. 响应式检查

- [ ] 桌面宽度下布局正常
- [ ] 窄屏下没有明显横向溢出
- [ ] 按钮和输入框没有重叠

### 23. 控制台检查

- [ ] 没有未处理异常
- [ ] 没有重复刷新的死循环迹象
- [ ] 没有显著 React key / hydration 警告

---

## 八、验收结论

当以下项目全部成立时，可以认为当前版本达到了本阶段目标：

- [ ] `npm run lint` 通过
- [ ] `npm run build` 通过
- [ ] 本地保存稳定
- [ ] 刷新稳定
- [ ] 导入导出可用
- [ ] 重置可用
- [ ] 七个模块基础 CRUD 可用
- [ ] 没有明显控制台错误

如果其中任何一项失败，应先修稳定性，再考虑继续加功能。
